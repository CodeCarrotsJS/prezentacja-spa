Single Page Applications

Sposób projektowania współczesnych stron internetowych, oparty na JavaScripcie.

Strony takie są szybkie, interaktywne i komunikują się z serwerem 'pod spodem'.

Zasada dziania klasycznej strony internetowej

  • wysyłamy request HTTP
  • serwer odpowiada przesyłając dane - HTML, CSS, JS
  • po kliknięciu w element aplikacji następuje kolejny request do serwera, który znów odpowiada...

Zasada działania SPA

  • wysyłamy request HTTP
  • serwer odpowiada przesyłając dane - HTML, CSS, JS
  • po kliknięciu w element aplikacji następuje kolejny request do serwera - serwer odpowiada jedynie danymi w formacie JSON, które są przetwarzane przez załadowane już pliki
  • dane od użytkownika do serwera też są wysyłane w formacie JSON i nie powodują przeładowania strony

SPA - zalety

  • lepszy UX
  • przyjaźniejsze dla urządzeń mobilnych
  • separacja zadań serwera i strony klienckiej

SPA - wady

  • cięższe do dobrego zaprojektowania
  • trudno indeksowalne
  • dziwne adresy
